Interview Prep & DSA Mechanics

Explain your Task Scheduler Optimization System.

Answer: This project is a task scheduling optimization system that resolves CPU/thread execution bottleneck issues. In real-world computing, jobs are loaded with strict deadlines and vary in execution length and importance (profit). When overloaded, processing jobs on a FIFO basis leads to system degradation. This system implements four scheduling algorithms: Greedy Deadline Job Scheduling, Priority Queue Scheduling, Earliest Deadline First (EDF), and Shortest Job First (SJF). It visualizes scheduling metrics in a Gantt-style timeline and showcases binary heap array index calculation mechanics.

How does a Binary Max-Heap operate, and what are its complexities?

Answer: A Binary Heap is a complete binary tree stored inside a contiguous array. In a Max-Heap, every parent node's value must be greater than or equal to its children's values. The index relationships are:

  • Parent of index $i$: Math.floor((i - 1) / 2)
  • Left child of index $i$: 2 * i + 1
  • Right child of index $i$: 2 * i + 2

Runtimes: Insertion is $O(\log n)$ by appending to the end and performing a bubble up operation. Extracting the maximum value is $O(\log n)$ by swapping the root with the last element, popping it, and performing a bubble down operation. Peeking is $O(1)$.

Why does the Greedy Deadline Scheduler produce the highest profit?

Answer: The Greedy Deadline Scheduler is an implementation of the classic Job Sequencing Problem with Deadlines. By sorting all tasks by profit in descending order, we iteratively choose the most valuable task first. To leave early time slots open for other tasks with tighter deadlines, we schedule each selected task in its latest possible free slot before its deadline. This greedy choice holds the optimal substructure property and yields the maximum possible total profit.
